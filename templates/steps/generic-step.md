# 通用产物步骤模板

> 定位：定义某类产物的生成规范，包括头部格式、章节结构、边界规则和审计要求。
>
> 使用方式：复制到 `.artifacts/steps/{type}-step.md`，根据项目需求修改。

---

## §1 产物头部标准

每份产物文件头部必须包含：

```markdown
# {产物名称}

**版本号**：v{主版本}.{补丁号}
**制定日期**：YYYY-MM-DD
**作者**：姓名

**上游文档**：
| 文档 | 类型 | 引用范围 |
|------|------|---------|
| {上游文档路径/名称} | 规范继承 / 需求输入 / 设计输入 | {具体章节/编号/范围} |
```

- `类型` 必须明确是**规范继承**还是**输入依赖**
- `引用范围` 必须具体到章节、编号或功能点，不能写"全文"

---

## §2 章节结构

| 章节 | 说明 | 必填 |
|------|------|------|
| §1 概述 | 背景、目标、范围 | 是 |
| §2 输入与约束 | 上游文档、全局契约、技术约束 | 是 |
| §3 核心内容 | 根据产物类型定义的具体内容 | 是 |
| §4 边界与例外 | 明确不做、例外场景、限制条件 | 否 |
| §5 验收标准 | 可验证的完成条件 | 是 |
| §6 附件 | 参考资料、链接 | 否 |

> 章节编号固定，确保下游产物引用稳定性。如无内容，保留标题并声明"跳过"。

---

## §3 产物边界

**本产物不写**：
- 已在上游定义的内容
- 属于下游职责的内容
- 超出当前产物类型范围的内容

**发现越界内容时**：标记并提示迁移，不在当前产物中默默兼容。

---

## §4 审计要求

产物生成后必须：

1. **机械检查**：运行 `audit-engine/cli.py --file <产物路径> --type <类型>`
2. **语义检查**：按 `checklists/{type}-checklist.md` 逐项判断
3. **审查**：通过 `review-before-acceptance` 审查
4. **收尾**：通过 `artifact-finishing` 做出四选一决策

---

## §5 变更传播

- 修改上游 → 扫描下游影响 → 询问是否级联修改
- 修改下游 → 读取上游 → 确认未超出边界
