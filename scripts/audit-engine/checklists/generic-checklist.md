# 通用语义检查清单

> 机械检查（格式、编号、表格）由 audit-engine 执行。以下为 AI 语义检查项。
> 使用方式：AI 读取本清单，逐项判断，输出固定格式表格。

---

## AI 执行纪律（不可违背）

1. **逐项回应**：必须回应本清单中的**每一项**，不得跳过、合并或用"等其他"概括。
2. **禁止减项**：如果检查项数量超过 **10** 项，或单份产物内容较多，**不得**自行减项。必须先输出分步计划，用户确认后执行。
3. **覆盖声明**：输出开头必须声明——应检查 N 项，已回应 N 项，遗漏 0 项（如有遗漏必须说明原因）。
4. **证据要求**：每项结论必须引用产物原文片段，禁止笼统写"符合要求"。
5. **上下文不足时**：如果当前上下文无法完成全部检查，**必须停止执行并请求拆分**，不得截断。

---

## 输出格式要求（AI 必须遵守）

每项输出必须包含 5 个字段：

| 字段 | 要求 |
|------|------|
| **检查项** | 复制本清单中的检查项编号和名称 |
| **结论** | 通过 / 不通过 / 存疑（三选一） |
| **证据** | **必须引用产物原文片段**，禁止笼统写"符合要求" |
| **判定依据** | 为什么是这个结论，引用本清单中的判定标准 |
| **建议** | 结论为"不通过"或"存疑"时给出具体修正方向 |

**禁止**：跳过检查项、合并多项为一个结论、无证据写"✅"。

---

## 上游一致性

| 编号 | 检查项 | 判定标准 |
|------|--------|---------|
| G-U-01 | 范围未越界 | **通过**：产物内容落在其职责范围内。**不通过**：存在超出边界的内容 |
| G-U-02 | 术语一致性 | **通过**：使用的业务术语与全局契约一致，同一概念无多个名称。**不通过**：发现术语别名混用 |
| G-U-03 | 引用完整性 | **通过**：头部声明的上游范围与实际内容中引用的范围一致（声明了就要覆盖，覆盖了就要声明） |

## 单产物内语义

| 编号 | 检查项 | 判定标准 |
|------|--------|---------|
| G-S-01 | 内容可验证 | **通过**：关键陈述有可量化的指标或可验证的条件。**不通过**：描述为"根据实际情况""视情况而定"等不可执行表述 |
| G-S-02 | 逻辑自洽 | **通过**：产物内部无矛盾陈述。**不通过**：同一概念在不同位置定义矛盾 |
| G-S-03 | 边界清晰 | **通过**：明确声明了"做什么"和"不做什么"。**不通过**：范围模糊，无法判断是否越界 |

## 边界维度

| 编号 | 检查项 | 判定标准 |
|------|--------|---------|
| G-B-01 | 无越界内容 | **通过**：产物中不包含属于上下游职责的内容。**不通过**：发现不属于当前产物类型的内容 |
