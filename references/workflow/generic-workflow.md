## §4 审计规则

### 4.1 机械检查

| 配置项 | 值 |
|--------|---|
| **审计引擎** | `scripts/audit-engine/cli.py` |
| **配置目录** | `.singularity/config/`（项目自定义）或 `scripts/audit-engine/config/`（插件默认） |
| **检查维度** | 格式、存在性、连续性、编号映射、契约追溯 |
| **通过标准** | 所有检查项通过（无警告/错误） |
| **失败处理** | 修复后必须重新执行完整机械检查 |

### 4.2 语义检查

| 配置项 | 值 |
|--------|---|
| **检查清单** | `checklists/{type-id}-checklist.md` |
| **执行者** | AI（需要业务上下文理解） |
| **通过标准** | 所有 L1 检查项通过，L2/L3 检查项无阻塞性问题 |
| **失败处理** | 修复后重新执行语义检查；如涉及格式/编号调整，追加机械检查 |

### 4.3 轮次控制

- 单份产物审计+修复最多 **3 轮**（含初次审计）
- 第 3 轮仍不通过 → 停止自动修复，向用户呈现剩余问题，请求人工决策

## §5 变更传播规则

### 5.1 上游变更传播

```
修改上游产物
    ↓
运行 dag-engine --impact {修改文件} --dir {产物目录}
    ↓
列出影响范围清单（直接影响 → 间接影响 → 无影响）
    ↓
询问用户是否级联修改
    ├─ 用户确认 → 同步修改所有受影响下游
    └─ 用户拒绝 → 在受影响下游标注 [待同步]
    ↓
全量重新审计
```

### 5.2 下游变更边界检查

```
修改下游产物
    ↓
读取所有上游产物
    ↓
逐条比对修改内容是否在上游定义范围内
    ├─ 未超出 → 直接修改
    └─ 超出 → 立即停止，先改上游
```
