# 证据先于断言（Evidence Before Claims）

## 概述

未经验证就声称完成是假设，不是事实。

**核心原则：** 永远先证据，后断言。

**违反本规则的字面要求就是违反本规则的精神。**

## 铁律

```
没有新鲜验证证据，不得声称完成
```

如果你在当前消息中还没运行验证，你就不能说它通过了。

## 门控函数

在声称任何状态或表达满意之前：

1. **识别**：什么命令、检查或证据能证明这个断言？
2. **运行**：执行**完整**的验证（新鲜的、完整的）
3. **读取**：读取完整输出，检查结果，统计失败或异常
4. **验证**：输出是否确认了断言？
   - 如果否：用证据说明实际状态
   - 如果是：附带证据做出断言
5. **只有在这之后**：做出断言

跳过任何一步 = 在假设，不是在验证

## 常见失败

| 断言 | 需要什么 | 不足够的 |
|------|---------|---------|
| 任务完成 | 每个需求都已满足的证据 | 感觉做完了，部分检查 |
| 问题已解决 | 原始症状不再复现 | 内容变了，假设已修 |
| 决策正确 | 证据支持所选选项 | 信心、直觉 |
| 分析准确 | 源数据与结论匹配 | 略读、凭记忆 |
| 计划已执行 | 每一步都有完成证据 | 智能体报告了「成功」 |
| 需求已满足 | 逐条对照规格清单检查 | 大致感觉对齐 |

## 危险信号 —— 停下

- 使用「应该」、「大概」、「看起来"
- 在验证之前就表达满意（「太好了！」「完美！」「搞定了！」）
- 在还没运行检查就要声明成功
- 相信之前运行的记忆
- 依赖部分验证
- 想「就这一次"
- 累了想收工
- **任何在没有运行验证的情况下暗示成功的措辞**

## 借口预防

| 借口 | 真相 |
|------|------|
| "现在应该能用了" | **运行**验证 |
| "我有信心" | 信心 ≠ 证据 |
| "就这一次" | 没有例外 |
| "部分检查够了" | 部分证明不了什么 |
| "换个说法规则就不适用了" | 精神高于字面 |
| "我累了" | 疲惫 ≠ 借口 |
| "智能体说成功了" | 独立验证 |
| "看起来不错" | 看起来 ≠ 已验证 |

## 关键模式

**完成工作时：**
```
✅ [运行完整验证] [看到：所有项通过] "工作完成——所有需求已验证"
❌ "工作完成"（没有运行验证）
```

**修复问题时：**
```
✅ [复现原始症状] [验证不再发生] "问题已解决——原始症状不再复现"
❌ "应该修好了"（没有复现检查）
```

**做决策时：**
```
✅ [根据标准评估选项] [确认所选选项满足标准] "决策：X——满足标准 A、B、C，有证据"
❌ "X 是最佳选择"（没有基于标准的评估）
```

## 必需子技能

> 注：本 skill 是结果验证的终点 skill，不调用任何同级或下游 Singularity skill（避免循环引用）。验证发现问题时，将问题交回给调用方处理。

如果验证发现**问题并未真正修复**（症状仍然存在或根因未消除）：
- **报告实际状态**：用证据说明问题仍在，将根因调查交回给调用方（systematic-problem-solving）
